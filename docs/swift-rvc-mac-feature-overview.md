# Swift RVC Mac Feature Overview

## 1. 这是什么

- 这是当前 `SwiftRVCMacClient` 的界面功能总览，按你在主界面里真的能看到的区域来解释，而不是按源码目录解释。
- 核心主线现在有四条：
  - `单文件变声`
  - `批量变声`
  - `实时变声`
  - `文本生成目标音色`
- 当前版本还额外集成了：
  - `UVR` 人声 / 伴奏分离
  - `TASK` 任务队列
  - `RES` 历史结果归档
  - `BG / LVL / MERGE` 背景声回混

## 2. 界面怎么读

### 2.1 顶部动作区

| 控件 | 用法 | 什么时候用 |
| --- | --- | --- |
| `BOOT` | 启动后端引擎 | 第一次打开 app 或引擎崩掉后重启 |
| `LINK` | 刷新后端连接并重新选择兼容实例 | 后端刚重启、怀疑前端还连着旧实例时 |
| `SYNC` | 重新扫描模型和索引目录 | 你刚放入新的 `.pth` / `.index` 后 |
| `UVR` | 打开人声 / 伴奏分离工具 | 想先把歌曲拆成 `vocals` 和 `instrumental` |
| `ONNX` | 打开 ONNX 相关工具 | 只在需要额外 ONNX 工具链时使用 |
| `CKPT` | 打开 checkpoint 工具 | 处理训练态 checkpoint、抽取部署模型时使用 |
| `AUDIO` | 刷新音频主机与输入输出设备 | 更换耳机、麦克风、虚拟声卡后 |
| `IN` | 打开输入来源面板 | 需要在音频文件输入和文本生成输入之间切换时 |
| `ASSET` | 打开资源相关工具 | 查看或整理工程资源时使用 |
| `HELP` | 打开帮助入口 | 忘了参数含义或流程时 |
| `LIVE` / `STOP` | 启动或停止实时变声 | 麦克风实时变声时使用 |
| `MASTER` | 只读主活动旋钮 | 看当前整体活动强度，不负责控制 |

### 2.2 PATCH BAY

这是“当前声音补丁”的核心区域，决定你正在把谁变成谁。

| 参数 | 作用 | 建议 |
| --- | --- | --- |
| `VOICE MODEL` | 当前加载的目标音色模型 | 每次只会有一个模型真正驻留内存 |
| `SPEAKER ID` | 多说话人模型的角色槽位 | 单说话人模型通常固定为 `0` |
| `F0 METHOD` | 音高提取算法 | 当前默认从 `CREPE` 开始，机器吃不消或想更稳时再退到 `RMVPE` |
| 区域右上角重置图标 | 把这一块恢复成默认参数 | 调乱了想快速回到默认值时 |

#### `VOICE MODEL`

- 你在这里选“目标音色”。
- 选择新模型时，客户端会先卸载旧模型，再加载新模型。
- 如果这里显示 `Choose target voice`，说明还没有真正加载任何目标模型。

#### `SPEAKER ID`

- 只对多说话人模型有意义。
- 如果一个模型有多个说话人，`0...N` 就代表不同槽位。
- 如果当前模型是单说话人模型，请保持 `0`。

#### `F0 METHOD`

- 它决定 app 怎样从原音频里提取“音高曲线”。
- 这是离线单文件、离线批量、实时变声三条链路共用的核心参数。
- 选错时，最容易出现的问题是：
  - 音高不稳
  - 低音区抖动
  - 高音区发虚
  - 运算太慢

### 2.3 ROUTE

这是实时链路的音频路由区，只影响 `LIVE`，不影响离线文件变声。

| 参数 | 作用 | 说明 |
| --- | --- | --- |
| `HOST` | 音频主机 API | 一般切换底层驱动或兼容性时才改 |
| `INPUT` | 输入设备 | 你的麦克风或虚拟输入 |
| `OUTPUT` | 输出设备 | 耳机、音箱或虚拟输出 |
| `MON` | 监听模式 | `VC` 听转换后声音，`INPUT` 听原始输入 |

### 2.4 REALTIME LAB

这是实时变声专属参数区，主要解决“延迟、稳不稳、噪声、断裂”。

| 参数 | 作用 | 怎么理解 |
| --- | --- | --- |
| `SR MODE` | 实时采样率来源 | `MODEL` 跟模型采样率，`DEVICE` 跟声卡采样率 |
| `BUFFER` | 额外缓冲时间 | 越大越稳，但延迟更高 |
| `CPU` | CPU 工作进程数 | 越高不一定越好，先按机器核心数试 |
| `GATE` | 输入门限 | 过滤底噪；越高越容易切掉轻声 |
| `FORMANT` | 共振峰偏移 | 微调音色明暗，不是升降调本身 |
| `WINDOW` | 实时分析窗口长度 | 大一点更稳，小一点更灵敏 |
| `FADE` | 块之间的交叉淡化长度 | 提高后通常能减少点击声 |
| `IN CLEAN` | 输入前降噪 | 先清理麦克风底噪 |
| `OUT CLEAN` | 输出后降噪 | 对转换后噪声再做抑制 |
| `SMOOTH` | Phase-vocoder 平滑 | 想让实时结果更顺时可尝试 |

### 2.5 PATCH CONFIG

这是离线变声的共享配置区。

| 参数 | 作用 | 说明 |
| --- | --- | --- |
| `PARAM BANK` | 当前底部推子正在控制哪套参数 | `Single` 控单文件，`Batch` 控批量 |
| `INDEX FILE` | 当前索引选择 | 可以自动、手选、外部覆盖、或不用索引 |

#### `PARAM BANK`

- 它不会改变模型，也不会改变实时链路。
- 它只决定底部推子当前是写入：
  - `单文件变声参数`
  - `批量变声参数`

#### `INDEX FILE`

- `Index` 是可选增强项，不是所有模型都必须有。
- 主要作用是帮模型保留更多目标音色特征。
- 经验上：
  - 模型声音已经很好时，可以先不加
  - 想让“更像目标角色”时，再尝试打开

### 2.6 FADERS

这一排竖推子是离线任务的主调参面板。它们受 `PARAM BANK` 控制。

| 缩写 | 全名 | 作用 |
| --- | --- | --- |
| `S-P / B-P` | Pitch | 升降调，单位半音 |
| `S-I / B-I` | Index | 索引参与强度 |
| `S-F / B-F` | Filter | 音高平滑滤波半径 |
| `S-R / B-R` | Resamp | 输出重采样目标 |
| `S-M / B-M` | RMS | 原始响度与转换响度混合比例 |
| `S-G / B-G` | Guard | 保护值，减少过度变声带来的嘶裂 |
| `QUE` | Queue | 只读，表示批处理输入是否已准备好 |
| `PLY` | Play | 只读，表示当前是否有可试听产物 |

### 2.7 底部动作区

| 按钮 | 作用 | 什么时候用 |
| --- | --- | --- |
| `PTH` | 打开模型权重目录 | 检查 `.pth` 是否放对了 |
| `IDX` | 打开索引目录 | 检查 `.index` 是否放对了 |
| `DIR` | 选择批处理输入目录 | 整文件夹一起跑 |
| `FILES` | 选择一个或多个输入文件 | 单文件和显式批量都能从这里进 |
| `TASK` | 打开任务队列面板 | 看当前任务状态、错误、输入输出 |
| `RES` | 打开历史结果面板 | 回看历史任务、试听、删除旧产物 |
| `OUT` | 选择批处理输出目录 | 批量任务输出位置 |
| `GO` | 启动单文件变声 | 当前有单个输入文件时 |
| `PLAY` | 试听当前产物 | 单文件转换完成后 |
| `OPEN` | 在 Finder 里打开当前产物 | 想直接看输出文件时 |
| `UNLD` | 卸载当前模型 | 释放模型内存时 |

### 2.8 波形区

波形屏不是单纯的装饰，它承担了试听和背景声回混。

| 控件 | 作用 | 说明 |
| --- | --- | --- |
| 播放 / 暂停按钮 | 播放当前产物 | 试听当前音频 |
| 进度条 | 跳转当前播放进度 | 可拖动 |
| `BG` / `BG ON` | 打开或关闭背景声回混试听 | 只有当前结果能追溯到 UVR 时才有意义 |
| `LVL` | 背景声占比 | 只影响背景声混入比例 |
| `MERGE` | 一键导出前景 + 背景合成版本 | 会生成新的合成产物并进入历史 |

#### 底部监视条

- 黑色监视面板底部会显示一排运行时指标。
- 其中 `BACKEND` 是当前实际连接的后端版本号，不是文档版本，也不是 app 版本。
- 如果你刚更新了后端：
  1. 先点 `LINK`
  2. 再看 `BACKEND` 是否变成预期版本
- 这个显示的目的就是防止前端误连到旧后端实例。

### 2.9 TASK

- `TASK` 是当前任务状态面板。
- 你能在这里看见：
  - 当前模型
  - 当前任务状态
  - 当前输入文件
  - 当前输出位置
  - 当前索引 / F0 / speaker
  - 当前错误信息

### 2.10 RES

- `RES` 是历史结果归档。
- 你可以按 `ALL / SINGLE / TEXT / BATCH / LIVE / UVR` 分类筛选。
- 每条记录会保存：
  - 时间
  - 模型
  - 输入
  - 输出
  - 存储目录
  - index / f0 / speaker
  - 文本任务的 `gender / tone / target match / 关键参数摘要`
  - 文本任务的阶段耗时摘要
  - 关联来源任务
  - UVR vocal / instrumental 关联
  - merged 产物
- 现在删除历史时会弹二次确认。

### 2.11 TEXT INPUT

`TEXT INPUT` 是通过 `IN` 面板进入的文本生成音频工作流。它的目标不是做一个泛化 TTS 实验室，而是：

`输入文本 -> 生成源语音 -> 套当前 RVC 模型 -> 尽量逼近目标音色`

#### 面板结构

| 区块 | 作用 | 你应该怎么理解 |
| --- | --- | --- |
| `1. GENDER` | 先锁定源语音的性别基线 | 先决定喂给 RVC 的原始声线方向，而不是直接改最终模型 |
| `2. TONE MODE` | 选择 `Preset` 或 `Custom` | 想要稳定结果用 `Preset`，想加一点可控语气用 `Custom` |
| `3. TONE` | 选择具体语气预设或输入自定义语气 | 每个 preset 都绑定了后端 ChatTTS 配置和一组推荐参数 |
| `4. TARGET MATCH` | 决定“像 RVC 模型音色”这件事要压多狠 | 当前默认是 `RVC Voice Priority` |
| `5. MAIN PARAMETERS` | 保留最影响贴模效果的主参数 | 不再暴露一大墙次要开关 |

#### `GENDER`

- `Female` / `Male` 先决定源语音的基础 formant 方向。
- 它会影响：
  - 可选女性或男性 tone preset
  - 自定义语气时的默认解释方向
  - 推荐 `transpose` 基线
- 现在有一条额外护栏：
  - 如果目标 RVC 模型被判断为女声，且你也选择了 `Female`
  - 那么默认 `transpose` 会被收回到 `0` 或更低
  - 也就是“不再为了女声而自动再升调一次”
- 只有在目标模型不是明确女声时，女性源语音才会继续按女性基线抬到更高区间。

#### `TONE MODE`

- `Preset`
  - 用已经调过的语气模板，优先给你一个更稳的起点
- `Custom`
  - 允许你输入如 `soft idol`、`deep serious`、`gentle narration`
  - 但后端会做“受约束解析”，不会把它放成开放式实验面板

#### `TONE`

- 女性预设

| 风格 | 效果说明 | 语速 | 适合什么内容 |
| --- | --- | --- | --- |
| `Sad Youth` | 带一点青春感的伤感，情绪轻，但不发飘 | 可单独调 | 青春遗憾、回忆感、轻伤感独白 |
| `Gentle` | 温和、柔和、不过度表演，适合平静表达 | 可单独调 | 安静对话、温柔口播、轻缓说明 |
| `Elegant Mature` | 更像御姐，控制感更强，甜度更低 | 可单独调 | 成熟台词、冷静表达、偏稳的女性角色 |
| `Bright Cheerful` | 活泼、开朗、明亮，开口更有精神 | 可单独调 | 元气句子、互动口播、开心表达 |
| `Natural` | 最普通、最均衡的女声起点 | 可单独调 | 默认试音、大多数普通文本 |
| `Heartbroken` | 更重的伤心和悲痛，尾音更脆弱，适合慢一点 | 可单独调 | 失恋、悲痛、情绪明显下坠的句子 |

- 男性预设

| 风格 | 效果说明 | 语速 | 适合什么内容 |
| --- | --- | --- | --- |
| `Warm Solid` | 中心稳定、厚度适中，最通用的男声基线 | 中 | 大多数普通文本、第一轮直接试音 |
| `Clear Lead` | 更亮、更靠前，年轻感和清晰度更强 | 快 | 年轻男声、清爽台词、需要更透一点的句子 |
| `Deep Anchor` | 更低、更厚、更沉，重心压得更下 | 慢 | 深男声模型、沉稳表达、压低重心的句子 |
| `Calm Narration` | 节奏稳，情绪克制，整体更偏旁白 | 中 | 解说、说明、平静叙述 |

- 每个 preset 不是单纯名字，它同时绑定：
  - ChatTTS 的 `speed / prompt / spk_emb / sampling`
  - 一组面向目标音色贴合的主参数默认值
- `RATE` 现在是主参数之一，直接控制源语音的快慢。
  - `Slow`: 更适合伤感、抒情、旁白
  - `Mid`: 作为新的通用默认，优先保字和稳定度
  - `Fast`: 只建议给明确需要活泼、跳跃、元气感的句子

#### 情绪 token 对照表

- 当前女性和男性 preset 的差异，不只是名字不同；后端会把不同 preset 映射到一组 `ChatTTS` 控制 token。
- 这些 token 主要控制的是 `prosody`，也就是：
  - 语速
  - 语气
  - 停顿
  - 口语感
  - 情绪起伏
- 它们不是最终音色本身。当前链路的目标是：
  - 前段 `TTS` 尽量只控制“怎么说”
  - 后段 `RVC` 尽量控制“像谁的音色”
- 现在女性 preset 也尽量共用更稳定的女性基线，不再靠随便切换底色种子去制造巨大音色差异。

| token | 主要控制什么 | 数值变高通常会怎样 | 数值变低通常会怎样 | 更适合什么风格 |
| --- | --- | --- | --- | --- |
| `[oral_x]` | 口语感、张口感、表达外放程度 | 更像在“说”，更外放，更有开口感，更容易显得活泼 | 更收、更像平读、更克制 | 活泼、青春、开朗、互动口播 |
| `[laugh_x]` | 笑意、轻快感、兴奋感 | 更亮、更跳、更有轻快和兴奋感，不一定真的在笑 | 更冷静、更平、更少轻快感 | 活泼、开朗、元气、偶像感 |
| `[break_x]` | 停顿、断句、呼吸感 | 更容易拉开句子、拖慢节奏、产生抒情或伤感感 | 更连贯、更冲、更适合明快表达 | 伤感、抒情、旁白、悲痛 |
| `[speed_x]` | 语速 | 更快、更紧、更有冲劲，但太高可能显得抢字 | 更慢、更舒展、更适合情绪句，但太低可能拖 | 活泼快句、正常对话、抒情慢句 |

#### 情绪 token 的直觉理解

| 组合倾向 | 听感结果 |
| --- | --- |
| `oral` 高 + `laugh` 高 + `break` 低 + `speed` 高 | 更活泼、更开朗、更像元气表达 |
| `oral` 中 + `laugh` 低 + `break` 中 + `speed` 中 | 更自然、最接近日常说话 |
| `oral` 低 + `laugh` 低 + `break` 高 + `speed` 中低 | 更温和、更柔、更慢，更容易进入抒情区间 |
| `oral` 低 + `laugh` 低 + `break` 很高 + `speed` 很低 | 更伤心、更悲痛、更脆弱，但也最容易拖和闷 |

#### 当前 6 个女性预设的大致 token 方向

| 风格 | token 倾向 | 听感重点 |
| --- | --- | --- |
| `Sad Youth` | `oral` 偏高、`laugh` 低、`break` 偏高、`speed` 中速偏快 | 青春感还在，但已经带伤感 |
| `Gentle` | `oral` 低、`laugh` 低、`break` 中高、`speed` 中速 | 温和、轻柔、克制，不应该闷成非人声 |
| `Elegant Mature` | `oral` 低到中、`laugh` 低、`break` 中、`speed` 中速 | 更稳、更控场、更成熟，但仍然要保持女声基础 |
| `Bright Cheerful` | `oral` 高、`laugh` 高、`break` 低、`speed` 高 | 活泼、开朗、明亮、跳跃感明显 |
| `Natural` | `oral` 中、`laugh` 低、`break` 中、`speed` 中速 | 最均衡、最像普通说话 |
| `Heartbroken` | `oral` 低、`laugh` 低、`break` 很高、`speed` 很低 | 更伤心、更悲痛、更脆弱，建议配 `Slow` |

#### 使用时怎么理解

- 如果你想要“更有风格”，优先先看 `oral / laugh / break / speed` 的组合，而不是一上来改很多 RVC 参数。
- 如果你想要“更像目标模型”，优先保持：
  - `TARGET MATCH = RVC Voice Priority`
  - `INDEX` 偏高
  - `F0 = CREPE`
- 如果出现“风格是对的，但音色不像模型”，优先调后段 `RVC` 参数。
- 如果出现“音色接近了，但说话方式不对”，优先换 `TONE` 或调 `RATE`。

#### `TARGET MATCH`

| 选项 | 作用 | 适合什么时候 |
| --- | --- | --- |
| `RVC Voice Priority` | 默认值。最优先追当前 RVC 模型音色 | 你想直接拿最像模型的结果时 |
| `Balanced Match` | 稍稳一点，减少过激参数 | 想保守一点时 |
| `Expressive Match` | 多保留一些说话表情 | 想要自然起伏多一点时 |
| `Custom Tone Lock` | 给自定义语气用，但仍优先保住音色贴合 | 写了 custom tone 时 |

- `Apply tuned defaults` 会把当前 `Gender + Tone + Target Match` 的推荐值整套重铺回去。

#### `MAIN PARAMETERS`

| 参数 | 作用 | 当前默认倾向 |
| --- | --- | --- |
| `SPEAKER` | 多说话人模型的角色槽位 | 单说话人模型保持 `0` |
| `RATE` | 控制源语音语速 | 当前通用默认改为 `Mid`，伤感/抒情建议优先用 `Slow` |
| `F0` | 音高提取算法 | 当前默认 `CREPE`，优先追细节和贴模 |
| `TRANSPOSE` | 升降调，单位半音 | 按 `Gender + Tone` 自动给推荐值 |
| `INDEX` | 索引参与强度 | 当前默认偏高，优先像目标模型 |
| `PROTECT` | 保护辅音和脆弱边缘 | 当前默认偏低，给模型更多覆盖空间 |
| `RMS MIX` | 保留原始响度轮廓的比例 | 默认适度压住源语音个性 |
| `FILTER` | 平滑音高抖动 | 深声线 / 叙述型预设通常更高 |
| `INDEX FILE` | 当前实际使用的索引文件 | 没指定时显示 `AUTO MATCH` |
| `TONE` | 当前实际生效的 tone 名称 | 用来确认 preset 或 custom 解析结果 |

#### 原始音频是怎么生成的

- 现在 text 任务不是“文字直接进 RVC”，而是：
  - `text`
  - `ChatTTS` 生成源语音
  - 保存 `tts-source.wav`
  - 再交给 `RVC` 做目标音色转换
- 这条 `tts-source.wav` 就是“变音前的中间产物”。
- 客户端会把它作为 `SRC WAV` 保留下来，方便你排查：
  - 是前段 `TTS` 就已经不对
  - 还是后段 `RVC` 才把结果带偏

#### 当前的停顿与换气策略

- 现在不会再对整段文字做过细的强制切碎。
- 主体策略是：
  - 优先按 `。！？；` 这类强标点处理大句边界
  - 逗号保留在句内，只给轻停顿
  - 换气只出现在句号类大句结束时
- 目的很直接：
  - 既保住整句上下文的情绪连续性
  - 又避免在词中间出现奇怪断气
- 如果你听到句尾和下句开头更自然，就是这层策略在生效。

#### 文本任务在 `TASK / RES` 里会多显示什么

- `TASK` 里会显示文本任务当前阶段和阶段耗时
- 目前阶段包括：
  - `Preparing`
  - `Load TTS`
  - `Generate Speech`
  - `Convert Voice`
  - `Finalize`
- `RES` 里会保留：
  - `SRC WAV`（变音前原始音频）
  - `gender`
  - `tone`
  - `target match`
  - 关键参数摘要
  - 阶段耗时摘要

## 3. F0 METHOD 详细说明

这是最容易把人看懵，但又最影响结果的一组参数。

| 选项 | 特色 | 优点 | 代价 / 风险 | 适合什么情况 |
| --- | --- | --- | --- | --- |
| `PM` | 最快的传统方法 | 速度最快，机器压力最小 | 容易在唱腔大、转音多时不稳 | 干净语音、小样快速试音 |
| `DIO` | 经典轻量型方法 | 比 `PM` 更稳一点，依然很轻 | 对复杂唱段仍可能发飘 | 普通说话声、轻量实时尝试 |
| `HARVEST` | 传统高质量路线 | 对低音、噪声、唱段通常更稳 | 明显更慢 | 唱歌、低音男声、复杂源音 |
| `CREPE` | 重型神经网络方法 | 音高细节丰富，表达力强 | CPU / GPU 压力最大 | 你愿意换算力去换细节时 |
| `RMVPE` | 当前稳妥回退方案 | 稳、准、鲁棒，综合表现很好 | 细节通常不如 `CREPE` 激进 | 机器吃不消或想更保守时 |
| `FCPE` | 轻量神经网络方法 | 比传统方法更顺，通常比 `CREPE` 省资源 | 不一定比 `RMVPE` 更稳 | 想要神经方法，但又不想太重 |

### 3.1 怎么选

1. 当前默认先用 `CREPE`，尤其是文本生成目标音色链路。
2. 如果机器吃不消，或你更想要稳妥结果，再退到 `RMVPE`。
3. 如果还想更省资源，再试 `DIO` 或 `PM`。
4. 如果是歌声、低音、底噪较多，优先试 `HARVEST`。
5. 如果想找一个介于“传统方法”和“重型神经方法”之间的折中，试 `FCPE`。

### 3.2 UI 里怎么看

- 现在在 `PATCH BAY -> F0 METHOD` 的菜单里，每个选项都会直接显示一句短说明。
- 你不用每次翻文档，菜单里就能直接看到：
  - 它偏快还是偏稳
  - 更适合说话声还是歌声
  - 会不会更吃 CPU / GPU
- 如果只是想先得到“更像模型”的结果，优先从 `CREPE` 开始。
- 如果想要更稳一点的 fallback，再切到 `RMVPE`。

## 4. 典型流程

### 4.1 单文件变声

1. `BOOT`
2. `SYNC`
3. 在 `VOICE MODEL` 里选目标音色
4. 选 `SPEAKER ID`
5. 选 `F0 METHOD`
6. 视情况选择 `INDEX FILE`
7. `FILES` 选单个输入音频
8. `GO`
9. 在波形区 `PLAY`
10. 如有 UVR 来源，可开 `BG` 试听背景声回混，再用 `MERGE`
11. 结果会写入统一 `storage/`，并出现在 `RES`

### 4.2 批量变声

1. 先完成 `VOICE MODEL / SPEAKER ID / F0 METHOD / INDEX FILE`
2. `PARAM BANK` 切到 `Batch`
3. `DIR` 选目录，或 `FILES` 选多个文件
4. `OUT` 选输出目录
5. 调整批量 faders
6. 运行批量任务
7. 在 `TASK` 看进度，在 `RES` 回看结果

### 4.3 实时变声

1. 加载模型
2. `AUDIO` 刷新设备
3. 设好 `HOST / INPUT / OUTPUT / MON`
4. 调 `REALTIME LAB`
5. 点 `LIVE`
6. 若爆音或断裂，先调 `BUFFER / WINDOW / FADE`
7. 若底噪重，先试 `GATE / IN CLEAN / OUT CLEAN`

### 4.4 文本生成目标音色

1. `BOOT`
2. `SYNC`
3. 选择 `VOICE MODEL`
4. 点 `IN` 打开 `TEXT INPUT`
5. 先选 `GENDER`
6. 再选 `TONE MODE`
7. 选 `TONE`
8. `TARGET MATCH` 保持 `RVC Voice Priority`
9. 如有需要微调 `RATE / F0 / TRANSPOSE / INDEX / PROTECT / RMS / FILTER`
10. 输入文本并点 `Use text input`
11. 回到主界面后点 `GO`
12. 在 `TASK` 看阶段进度和耗时，在 `RES` 回看结果和 `SRC WAV`

## 5. 调参建议

- 文本结果不像目标模型：先保持 `RVC Voice Priority`，再提高 `INDEX`，必要时检查 `INDEX FILE`
- 声音不稳：先换 `F0 METHOD`，优先 `RMVPE` 或 `HARVEST`
- 嘶裂或辅音怪：提高 `GUARD`
- 响度不自然：调 `RMS`
- 实时爆音：提高 `BUFFER`，再看 `WINDOW / FADE`
- 实时延迟太高：降低 `BUFFER` 和 `WINDOW`，但要接受稳定性下降

## 6. 参数调高 / 调低会怎样

这一节专门回答“这个参数往左拉和往右拉分别会发生什么”。

### 6.1 离线参数

#### `PITCH`

- 调低：
  - 声音更低、更厚
  - 容易更接近低音角色
  - 拉太多时会闷、浑、失真
- 调高：
  - 声音更高、更亮
  - 更容易接近高音、少女感、少年感
  - 拉太多时会尖、薄、假

#### `INDEX`

- 调低：
  - 更接近模型直接推理结果
  - 通常更干净、更稳
  - 但目标角色特征可能没那么像
- 调高：
  - 目标音色特征更明显
  - 常常更“像角色”
  - 如果 index 不匹配，容易出现金属感、毛刺、发脆

#### `FILTER`

- 调低：
  - 保留更多原始 pitch 细节
  - 转音、颤音更灵敏
  - 但不稳时也更容易抖
- 调高：
  - pitch 更平滑
  - 抖动和突刺更少
  - 太高会把细节磨平，唱腔容易变钝

#### `RESAMP`

- 调低或接近 `0`：
  - 尽量保留当前输出采样率
  - 少一次额外重采样
  - 通常更自然
- 调高到明确目标值：
  - 输出更容易适配某些播放器或下游软件
  - 但会增加一次重采样过程
  - 设得不合理时可能让高频更硬或更糊

#### `RMS`

- 调低：
  - 更像纯转换结果自己的动态
  - 有时更“像模型”
  - 但响度起伏可能不够贴原曲
- 调高：
  - 更保留原始输入的响度轮廓
  - 通常和伴奏融合更自然
  - 太高时可能让转换结果的表情被压回原音

#### `GUARD`

- 调低：
  - 转换更激进
  - 目标音色更明显
  - 容易让辅音、齿音、气声炸开
- 调高：
  - 保留更多原音边缘和辅音细节
  - 更容易抑制嘶裂、发脆、过度变声
  - 太高时会让结果“不够像目标角色”

### 6.2 实时参数

#### `BUFFER`

- 调低：
  - 延迟更低
  - 说话反馈更跟手
  - 容易爆音、卡顿、掉块
- 调高：
  - 更稳
  - 爆音和断裂更少
  - 但延迟更明显

#### `CPU`

- 调低：
  - 资源占用更可控
  - 对轻任务或低功耗机器更稳
  - 但重负载下吞吐可能不够
- 调高：
  - 可能提高处理吞吐
  - 在多核机器上有机会更稳
  - 太高时也可能因为调度开销反而变差

#### `GATE`

- 调低：
  - 轻声、气声、尾音更容易保留下来
  - 房间底噪也更容易进来
- 调高：
  - 更能压住环境噪声
  - 但轻声、尾音、字头字尾可能被切掉

#### `FORMANT`

- 调低：
  - 音色更暗、更厚、更靠后
  - 太低时容易闷
- 调高：
  - 音色更亮、更薄、更靠前
  - 太高时容易尖和假

#### `WINDOW`

- 调低：
  - 响应更快
  - 但更容易 wobble 或不稳
- 调高：
  - 更平稳
  - 但延迟更高，也可能让快节奏细节变糊

#### `FADE`

- 调低：
  - 边缘更利落
  - 但块切换时更容易有点击声
- 调高：
  - 更能平滑块之间的衔接
  - 太高时会让起音和攻击感变软

### 6.3 背景声回混参数

#### `LVL`

- 调低：
  - 背景伴奏更弱
  - 更容易听清当前变声主唱
  - 适合单独检查人声质量
- 调高：
  - 背景伴奏更强
  - 更接近完整歌曲的听感
  - 太高时容易把主唱盖住

## 7. 当前输出和历史存储

- 新任务默认统一写到仓库根目录下的 `storage/`
- 不再以 `engine/opt` 作为主输出目录
- `UVR` 会保留：
  - `vocals`
  - `instrumental`
- 单文件变声如果来自某次 `UVR` 的 vocal，`RES` 会记录：
  - `SOURCE`
  - `SRC VOC`
  - `SRC INS`
- `MERGE` 产生的合成文件也会写回对应任务目录，并进入 `RES`

## 8. 一句话记住

- `PATCH BAY` 决定“用谁的声音”
- `ROUTE + REALTIME LAB` 决定“实时怎么跑”
- `PATCH CONFIG + FADERS` 决定“离线参数怎么调”
- `TASK` 看当前
- `RES` 看历史
- `BG / LVL / MERGE` 负责把 UVR 的背景声重新合回来
